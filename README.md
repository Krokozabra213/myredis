Redis<br>
├── String → "hello" Кеш, счётчики, сессии<br>
├── Hash → {name: "John", age: 30} Объекты (профиль пользователя)<br>
├── List → [msg1, msg2, msg3] Очереди, последние N элементов<br>
├── Set → {user1, user2, user3} Уникальные значения, теги<br>
├── Sorted Set → {user1: 100, user2: 200} Рейтинги, лидерборды<br>
├── Stream → Лог событий Очереди сообщений (аналог Kafka)<br>
├── HyperLogLog → Приблизительный COUNT Подсчёт уникальных посетителей<br>
└── Bitmap → 0110110100 Флаги, онлайн-статус<br>

Cache-Aside (Ленивая загрузка)<br>
Самый популярный паттерн. Приложение само управляет кешем.<br>

Чтение:<br>

1. Проверяем кеш<br>
2. Если есть → возвращаем<br>
3. Если нет → читаем из БД → кладём в кеш → возвращаем<br>

Запись:<br>

1. Пишем в БД<br>
2. Удаляем из кеша (НЕ обновляем!)<br>

Write-Through (Сквозная запись)
Кеш всегда синхронизирован с БД. Запись идёт через кеш.

Клиент → БД → Кеш

Сначала пишем в БД (источник правды), потом синхронно обновляем кеш.
Читаем всегда из кеша
